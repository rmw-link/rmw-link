# 对等网通讯协议设计

在构建软件之前，先要定义通讯协议。

我们的目标是实现一个对等网络 ：在对等网络架构中，人人平等，无人垄断数据。

# 身份秘钥

加入人民网络前，首先要生成身份秘钥。

身份秘钥是Ed25519密钥对。

首次启动时，客户端会自动生成一个密钥对。

不同设备可以导入同一秘钥，也就是说，你的平板、手机、电脑可以同时在线。

如果身份秘钥丢失了，只能放弃原有的账户，重新生成一个新的身份。

所以，务必对秘钥做好备份。

之后，用户可以设置网络昵称。昵称可以修改，但身份秘钥永远不变。

# 建立连接

假设有两个节点：A 和 B （一个节点就是一个在线的人民网设备）。

A要和B建立首次连接。

在创建连接之前，A会先发送一个字节的UDP包进行PING。

包的内容为 0x01 。

B收到了内容为 0x01 的UDP包，就会响应一个空的UDP包标识自己在线。

我们使用UDP作为底层的通讯协议，方便打洞和内网穿透，来实现节点之前互联互通。

这第一步，目的用来探测对方是否在线。

我们将探测在线的包设计的足够小，目的是方便进行内网打洞时候的进行端口扫描。

同时，我们保证建立连接之前，发送的包比响应的包尺寸更大（发送1个字节，响应0个字节），如此可以有效避免恶意节点制造UDP反射放大攻击。

enum Cmd { Ping = 1, Pong = 2 }
